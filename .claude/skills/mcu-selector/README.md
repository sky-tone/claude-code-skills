# MCU Selector — MCU 平台选型路线图技能

> 🎯 一键生成专业的 MCU 选型分析报告 PPT，从需求分析到选型决策全流程覆盖。

---

## 📋 功能概述

MCU Selector 是基于 Claude Code Skill 架构设计的 MCU 选型辅助技能，融合了资深嵌入式架构师的选型经验，可以：

1. **需求智能解析** — 从自然语言中提取选型关键约束
2. **候选方案筛选** — 基于全球主流 MCU 平台知识库，筛选 3-5 个候选方案
3. **多维度评估** — 八维度加权评分：性能 / 外设 / 功耗 / 成本 / 生态 / 供应链 / 可扩展性 / 认证
4. **可视化报告** — 使用 python-pptx 输出 25-35 页专业选型分析 PPT

---

## 🚀 使用方法

### 基本用法

```
帮我做一个 MCU 选型分析，用于 WiFi 智能插座，需要 WiFi + BLE，成本控制在 ¥5 以内
```

### 带参数用法

```
MCU选型: 工业温度传感器 --budget ¥3 --volume 100K --priority cost
```

### 详细需求描述

```
项目需求：
- 应用：三相电机 FOC 控制器
- 核心功能：三相 PWM、双路 ADC 同步采样、CAN-FD 通信
- 性能：PWM 频率 ≥ 20kHz，ADC 采样率 ≥ 1Msps
- 功耗：无特殊要求（工业供电）
- 成本目标：MCU ≤ ¥10 (10K pcs)
- 温度：-40°C ~ +85°C
- 团队经验：熟悉 STM32，也接受其他平台
- 量产规模：预计年产 5 万台
请帮我做选型分析并输出 PPT 报告
```

## 📊 报告结构

生成的 PPT 包含 **8 个章节，约 33 页**：

| 章节 | 页数 | 包含页面 |
|------|------|---------|
| 01 需求分析 | 4页 | 项目背景、关键约束、外设清单 |
| 02 市场调研 | 2页 | 可选平台概览、系统架构框图 |
| 03 候选方案 | 5页 | 每个方案一页：规格、优劣、生态 |
| 04 对比评估 | 4页 | 参数表、雷达图、评分矩阵、深度对比 |
| 05 成本分析 | 2页 | BOM成本影响、关键数据 |
| 06 风险评估 | 2页 | 风险矩阵、缓解措施 |
| 07 升级路线 | 2页 | Pin兼容路线图、项目里程碑 |
| 08 结论建议 | 2页 | 推荐方案、后续行动项 |

## 🎨 专用页面布局

除通用布局（封面 / 目录 / 章节 / 内容 / 卡片 / 时间轴 / 总结 / 结束页）外，MCU Selector 新增以下专用布局：

| 布局 | 函数 | 用途 |
|------|------|------|
| MCU 详情页 | `make_mcu_detail_slide()` | 单个 MCU 的完整信息展示 |
| 雷达图页 | `make_radar_chart_slide()` | 多维度评估分数可视化 |
| 评分矩阵页 | `make_scoring_matrix_slide()` | 加权评分表格 + 排名 |
| 升级路线图页 | `make_upgrade_roadmap_slide()` | Pin 兼容升级路径 |
| 外设匹配页 | `make_peripheral_match_slide()` | 需求 vs 方案 ✅/❌ 矩阵 |

## 💡 设计理念

本技能借鉴了 Claude Code 源码的架构思路：

1. **分阶段执行** — Phase 0~7 流水线式处理，每阶段职责清晰
2. **知识库驱动** — 内置 MCU 平台图谱 + 场景速查表，确保推荐准确
3. **模板化输出** — 标准化报告结构，可复用布局函数
4. **质量闭环** — 内置检查清单，确保选型报告的工程严谨性
5. **可扩展** — 新增厂商/系列只需更新知识库，不影响核心逻辑
