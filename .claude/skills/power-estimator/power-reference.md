## 二、BLE 广播功耗计算公式

```
BLE 广播平均电流 = I_adv_event × t_adv_event / T_adv_interval

其中:
  I_adv_event  = 广播事件电流 (3个广告信道) ≈ TX电流
  t_adv_event  = 单次广播事件时间 ≈ 1.2~3 ms (取决于广播数据长度)
  T_adv_interval = 广播间隔 (100ms ~ 10.24s)

示例 (nRF52832, 0dBm, 1s间隔):
  I_avg = 5.3mA × 1.5ms / 1000ms = 7.95 μA
  加上系统休眠电流: 7.95 + 1.9 = 9.85 μA
```

### BLE 连接功耗计算

```
BLE 连接平均电流 = I_conn_event × t_conn_event / T_conn_interval

其中:
  I_conn_event    = TX+RX 交替电流 ≈ 5~8 mA (均值)
  t_conn_event    = 连接事件时间 ≈ 2~5 ms (取决于数据量)
  T_conn_interval = 连接间隔 (7.5ms ~ 4s)
  
  如果使用 Slave Latency = N:
  等效间隔 = T_conn_interval × (N + 1)
```

## 三、WiFi 功耗计算参考

### WiFi 传输一次数据的能量

```
E_transmit = E_wakeup + E_association + E_data + E_ack

典型时间分解 (ESP32-C3, 发送100字节):
  唤醒+初始化:  10ms × 30mA  = 300 μA·s
  WiFi关联:     200ms × 90mA = 18000 μA·s (已连接则跳过)
  数据发送:     5ms × 320mA  = 1600 μA·s
  等待ACK:      10ms × 90mA  = 900 μA·s
  总计 ≈ 20,800 μA·s (首次) 或 2,800 μA·s (已关联)
```

### WiFi 周期上报功耗

```
如果每 T 秒上报一次:
  I_avg_wifi = E_transmit / T + I_modem_sleep

示例 (ESP32-C3, 每60s上报, 保持连接):
  I_avg = 2800 / 60 + 15000 = 15047 μA ≈ 15 mA (Modem Sleep 主导)
  
示例 (ESP32-C3, 每60s上报, Light Sleep):
  I_avg = 20800 / 60 + 130 = 477 μA (每次需重连)
```

## 四、电源路径效率计算

### LDO 效率

```
η_LDO = V_out / V_in × 100%

示例: 3.7V LiPo → 3.3V LDO
  η = 3.3 / 3.7 = 89.2%
  若系统消耗 10mA, 电池实际消耗 = 10 / 0.892 = 11.2mA
  LDO 自身消耗 = I_quiescent (通常 1~100μA)
```

### DC-DC 效率

```
η_DCDC = f(Vin, Vout, Iout)  — 查效率曲线图

典型值:
  满载 (>10mA):  85~95%
  轻载 (1mA):    70~85%
  极轻载 (<100μA): 40~70% (PFM模式更好)

示例: 3.7V LiPo → 3.3V Buck (η=90%)
  若系统消耗 10mA, 电池实际消耗 = 10 × 3.3 / (3.7 × 0.90) = 9.9mA
  DC-DC 自身 Iq = 5~50μA (额外消耗)
```

### 多电压域功耗换算

```
将所有电流折算到电池端:
  I_battery = Σ(I_domain_i × V_domain_i / (V_battery × η_path_i))

示例:
  1.8V 域消耗 5mA (通过 Buck, η=85%):  I_bat = 5×1.8/(3.7×0.85) = 2.86mA
  3.3V 域消耗 10mA (通过 LDO):          I_bat = 10×3.3/(3.7×1.0)  = 8.92mA
  电池总电流 = 2.86 + 8.92 + Iq = 11.78 + Iq mA
```

## 五、温度对功耗的影响

```
漏电流温度系数 (经验公式):
  I_leak(T) = I_leak(25°C) × 2^((T-25)/10)
  
  即: 每升高 10°C, 漏电流约翻倍

示例:
  25°C 时 Standby = 1μA
  85°C 时 Standby = 1 × 2^((85-25)/10) = 1 × 2^6 = 64 μA

影响范围:
  Stop/Standby/Shutdown 模式影响显著 (数量级变化)
  Run 模式影响较小 (通常 <20%)
```

## 六、CR2032 实际使用注意事项

```
CR2032 内阻模型:
  新电池内阻 ≈ 15~30Ω
  半放电后 ≈ 30~60Ω
  近放完 ≈ 100~200Ω

电压跌落 = I_pulse × R_internal

示例:
  脉冲 10mA 时: ΔV = 10mA × 30Ω = 0.3V
  → 电池3.0V - 0.3V = 2.7V (可能导致MCU复位!)

设计建议:
  1. 脉冲电流 ≤ 5mA (推荐)
  2. 加 100μF~470μF 旁路电容平滑脉冲
  3. MCU 工作电压应支持到 2.0V 以下
  4. 考虑使用超级电容缓冲 (0.1F~1F)
```

## 八、功耗预估常见坑

| 坑点 | 说明 | 避免方法 |
|------|------|---------|
| **只看 MCU 典型值** | Datasheet 典型值是理想条件, 实际偏高 | 使用最大值或+30%余量 |
| **忘记外设电流** | 上拉电阻、LED指示、电平转换等 | 逐条审查BOM中每个器件 |
| **忽略电源路径损耗** | LDO Iq, DC-DC轻载效率, 分压器漏电 | 显式计算电源管理消耗 |
| **睡眠态浮空引脚** | GPIO浮空导致 μA~mA 级漏电 | 全部未用引脚配置为模拟或上/下拉 |
| **唤醒尖峰遗漏** | 唤醒瞬间电流大且MCU尚未配置低功耗 | 计入唤醒时间×唤醒电流 |
| **Debug 口未断开** | SWD/JTAG 引脚持续耗电 | 量产时禁用调试接口 |
| **CR2032 过流** | 脉冲电流过大导致电压跌落复位 | 加旁路电容, 限制脉冲<5mA |
| **温度影响忽略** | 高温漏电可增加 10~100x | 在最高工作温度下评估 |
| **自放电遗漏** | 碱性电池自放电 ~3%/月 | 长寿命场景必须纳入计算 |
| **电池老化** | 锂电池2年后容量衰减 10~20% | 使用 80% 有效容量计算 |
