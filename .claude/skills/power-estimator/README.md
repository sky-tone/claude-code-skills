# Power Estimator — 嵌入式系统功耗预估分析技能

## 概述

Power Estimator 是一个专业的嵌入式系统功耗建模与分析技能，能够根据硬件方案自动进行多场景功耗预估、电池寿命计算、优化建议生成，并输出完整的 PPT 报告。

## 使用方法

### 基本调用

```
请帮我做功耗预估分析：

系统：BLE 体温贴片
MCU：nRF52832
外设：NTC温度传感器, BLE 每10秒广播一次
电池：CR2032 (220mAh)
目标寿命：>1年
```

### 高级调用 — 多场景分析

```
功耗分析：

产品：智能门锁
MCU：STM32L4R5 @80MHz
外设：
  - 电容式触摸键盘 (待机扫描+活跃输入)
  - 电机驱动 (开锁3秒)
  - BLE模块 (连接管理)
  - NFC读卡器 (被动唤醒)
电池：4节AA碱性电池 (串联6V, 容量2800mAh)
供电方案：6V → Buck 3.3V (η=88%), LDO 1.8V
工作场景：每天开锁10次, BLE保持连接8小时
环境温度：-10°C ~ 45°C
目标寿命：>18个月
```

### 对比分析

```
请对比以下两个MCU方案的功耗：

方案A: STM32L4 + SX1276 (LoRa)
方案B: STM32WL (集成LoRa)

应用: 每小时上报一次温湿度
电池: 3.6V ER14505 (2600mAh)
```

## 6个专用PPT布局函数

| 函数 | 用途 | 视觉效果 |
|------|------|---------|
| `make_scenario_overview_slide()` | 场景时间线总览 | 时间轴+对数尺度电流柱 |

| `make_optimization_slide()` | 优化建议表 | 优先级+节省量+复杂度矩阵 |

## 设计理念

1. **数据驱动** — 不猜测，每个数值都有出处
2. **场景化建模** — 不是单一平均值，而是多场景加权
3. **工程余量** — 典型值×1.3，容量×0.8，确保可靠
4. **可视化优先** — 瀑布图、饼图、敏感度图让数据说话
5. **可操作建议** — 不只是分析，更给出具体优化路径
